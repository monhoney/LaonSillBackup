스크립트를 실행하면 2가지 PATH로 비교하는데 먼저 1번 파일 기준으로 2번 파일과 비교, 그리고 다시 2번 파일 기준으로 1번 파일과 비교합니다.
각 PATH에서는 기준이 되는 파일에 있는 속성이 비교되는 파일에 존재하는 지, 존재한다면 같은 지를 비교 합니다.

예를 들어 기준이 되는 파일의 모든 속성들에 대해 비교되는 파일이 모두 같은 값을 갖고 있다면 해당 비교는 통과하게 됩니다.
하지만 비교되는 파일에 추가 속성이 있어도 이 테스트를 통과하기 때문에 첫번째 PATH에서 비교 파일을 다시 기준 파일로 하여 진행하게 됩니다.

이 테스트는 TRAIN / TEST / UNION 파일의 속성을 고려하여 처리합니다.


현재 SOOSEMI의 NETWORKDEF들에 대해 테스트 하였고 큰 문제는 없어 보입니다만 추가적인 버그 수정이 필요할 수 있는 상황입니다.
